# SoChi BLOCKS ガバナンス ― ゆるっと版 / Lightweight Governance

> **まだ小規模なプロジェクトなので、最小限・シンプルな運営ルールだけ!**
> We keep our rules lightweight because SoChi BLOCKS is still a tiny project.

---

## 目的 / Why We Have This

* 迷ったときの“決め方”を共有し、開発をスムーズに進める。
* ルールを最小限にして、まずはアウトプットを優先する。

## ロール / Roles (超ざっくり)

| 役割               | 主な権限 / What they do      |
| ---------------- | ------------------------ |
| **Project Lead** | リポジトリの最終責任者。方向性を決める。     |
| **Maintainer**   | PR レビュー・マージ権限。2 人いれば心強い。 |
| **Contributor**  | Issue / PR を投げるみんな。      |

> **昇格基準**：一定期間活発に貢献したら Lead が Maintainer に招待します。まずは楽しく！

## 意思決定 / How We Decide

1. **まず Issue / Discussion を立てる**

   * 質問・提案・仕様変更なんでも OK。
2. **ゆるふわコンセンサス**

   * 3 日（72 時間）異論がなければ採用。
3. **異論が出たら**

   * コメント欄で合意形成を試みる。
   * 収束しなければ **Project Lead が最終判断**。

> 👍🏽 **投票フォームなどは不要**。リアクション絵文字でも OK！

## リリース / Releases

* ブランチ運用：`main` = 安定版、`dev` = 次リリース候補。
* タグは `vX.Y.Z`（SemVer）。
* `main` にマージされたら即タグ cut & GitHub Release するかは Lead の気分次第🥳。

## ミーティング / Meetings

* **定例なし**。必要に応じて Slack/Discord で音声つなぐ。
* 雑談歓迎。コーヒー片手にどうぞ。

## 行動規範 / Code of Conduct

* みんな仲良く！困ったら [CoC](./CODE_OF_CONDUCT.md) に沿って対応。

## このドキュメントの変更 / Changing This Doc

* PR を出して 72 時間待って誰も反対しなければマージ。
* 大幅変更は Project Lead の承認をもらおう。
